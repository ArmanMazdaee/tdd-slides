آرمان مزدائی

آزمایش واحد

یک روش آزمایش، برای آزمودن واحدهای کوچکی از برنامه

یک مثال

import unittest

class TestStringMethods(unittest.TestCase):

    def test_isupper(self):
        self.assertTrue('FOO'.isupper())
        self.assertFalse('Foo'.isupper())

    def test_split(self):
        s = 'hello world'
        self.assertEqual(s.split(), ['hello', 'world'])
        # check that s.split fails when the separator is not a string
        with self.assertRaises(TypeError):
            s.split(2)

if __name__ == '__main__':
    unittest.main()

مزایا

  • زودتر پیدا شدن مشکلات

  • آسان‌تر کردن تغییرات آتی

  • خواناتر شدن کد

معایب

  • غیر واقعی بودن

  • چک نکردن ارتباط با سایر بخش‌ها

  • پوشش ندادن همه حالات

TDD

  • یک نوع متولوژی توسعه نرم‌افزار هست

  • توسط کنت بک به صورت رسمی در ۲۰۰۳ معرفی شد

  • جزئی از برنامه‌سازی مفرط هست

مراحل

اشتباهات رایج

  • نوشتن تعداد زیادی آزمایش در هر مرحله

  • نوشتن آزمایش‌های بزرگ

  • نوشتن کدی که کاری بجز آزمایش طرح شده می‌کنند

مزایا

  • افزایش بازده برنامه‌نویس

  • آسان کردن رفع ایرادات برنامه

  • بالابردن کیفیت کد

  • بالابردن خوانایی کد

  • بهبود طراحی برنامه

معایب

  • افزایش زمان تولید در ابتدا

  • نیاز به مهارت

  • چک نکردن ارتباط بخش‌های مختلف

ابزارها

  • کتابخانه‌های آزمایش واحد

  • اجراکننده آزمایش‌ها

  • کتابخانه‌های تقلد کننده
